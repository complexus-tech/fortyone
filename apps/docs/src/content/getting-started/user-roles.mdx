# User Roles and Permissions

The system provides different roles with varying levels of access to features and data.

## Administrator

Has full access to all features and settings. Can manage users, teams, and projects across the entire system.

## Team Lead

Can create and manage projects, add team members, and configure team settings. Has access to team analytics and reports.

## Member

Can create and manage stories, participate in sprints, and collaborate with team members.

## Observer

Has read-only access to projects. Useful for stakeholders who need to stay informed but don't directly contribute to the work.
